# Trip (여행방·일정) 비즈니스 규칙

> NotebookLM 기획 자료 정리본. UI 상세는 `design/figma-wireframe-v1.md` 참고.

| 규칙 ID | 규칙명 | 조건 | 동작 | 위반 시 (에러/제약) |
| :--- | :--- | :--- | :--- | :--- |
| **BR-TRIP-001** | 여행방 생성 필수 정보 | 방장이 여행방을 생성할 때 | 여행방 이름, 여행 희망 기간(범위), 희망 여행 일수(n박 m일), 참여 인원을 반드시 입력해야 함 | 생성 버튼 비활성화 또는 필수 입력 안내 노출 |
| **BR-TRIP-002** | 일정 응답 단위 | 참여자가 자신의 일정을 입력/수정할 때 | 날짜별로 오전, 오후, 저녁 세 단위로 구분하여 입력 가능해야 함 | 단위 미선택 시 저장 불가 |
| **BR-TRIP-003** | 일정 상태 정의 | 참여자가 시간대별 상태를 선택할 때 | 가능, 불가능(일정 있음), 미정(확정 전) 중 하나를 선택해야 함 | `[기획 자료에 없음]` |
| **BR-TRIP-004** | 개인 일정 프라이버시 | 타 참여자의 일정을 조회할 때 | 개인 일정의 제목이나 상세 내용은 비공개이며, 가능/불가/미정 상태값만 노출해야 함 | 상세 내용 노출 차단 |
| **BR-TRIP-005** | 추천 알고리즘 우선순위 | 시스템이 후보 일정을 산출할 때 | 1. 불가능 인원 최소화(참여 인원 극대화), 2. 연차 사용 최소화, 3. 불확실 일정(미정) 최소화 순으로 점수 계산 | 점수가 낮은 일정은 TOP 3 후보에서 제외 |
| **BR-TRIP-006** | 연차 조건 설정 | 사용자가 근무/연차 환경을 설정할 때 | 여행 당 최대 사용 가능 연차 일수, 연차 사전 신청 기간, 반차 사용 가능 여부를 설정할 수 있음 | 설정 범위 초과 입력 시 저장 거부 (최대 일수 등은 `[미정]`) |
| **BR-TRIP-007** | 일정 확정 권한 | 여행방의 최종 일정을 결정할 때 | 방장(총대)이 추천 후보 중 하나를 선택하거나 직접 날짜를 입력하여 확정함 | 일반 참여자는 확정 권한 없음 |
| **BR-TRIP-008** | 희망 여행 일수 제약 | 여행방 생성 시 희망 여행 일수를 설정할 때 | 설정된 여행 희망 기간(전체 범위)보다 희망 여행 일수가 더 길 수 없음 `[제안]` | 입력 단계에서 검증 에러 발생 |

## 기획 메모 (NotebookLM)

### `[미정]` 규칙

- BR-TRIP-003: 일정 상태 미선택 시 세부 시스템 처리
- BR-TRIP-006: 연차 일수 등 숫자 필드 최대/최소 임계값

### 기획 확인 필요

- **BR-TRIP-008**: 전체 범위 vs 여행 일수 검증 로직 확정
- **BR-USER-002** (`user.md`): 웹 참여 시 로그인 필수 여부·기능 제한 범위
