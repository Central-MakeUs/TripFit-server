# User (사용자·권한) 비즈니스 규칙

> NotebookLM 기획 자료 정리본.

| 규칙 ID | 규칙명 | 조건 | 동작 | 위반 시 (에러/제약) |
| :--- | :--- | :--- | :--- | :--- |
| **BR-USER-001** | 방장(총대) 인증 요구사항 | 사용자가 여행방을 생성하려 할 때 | 반드시 회원가입 및 소셜 로그인을 완료한 상태여야 함 | 로그인 화면으로 리다이렉트 |
| **BR-USER-002** | 참여자 진입 장벽 완화 | 참여자가 초대 링크를 통해 진입할 때 | 앱 설치 없이도 웹(또는 딥링크)을 통해 일정 응답 플로우에 진입할 수 있어야 함 | 앱 설치 강제 안 함 (기능 제한은 `[미정]`) |
| **BR-USER-003** | 소셜 계정 연동 | 사용자가 설정에서 계정을 관리할 때 | 카카오, 구글 등 소셜 계정 연동 정보를 관리할 수 있음 | 연동 해제 시 경고 문구 노출 |

## 보조 규칙 (Figma·PRD)

| 규칙 ID | 규칙명 | 조건 | 동작 | 위반 시 |
| :--- | :--- | :--- | :--- | :--- |
| **BR-USER-004** | 회원 탈퇴 | 사용자가 탈퇴 요청 시 | 확인 후 탈퇴; 내가 만든 여행방 데이터 삭제 (Figma 카피) | `[미정]` 진행중 여행방 존재 시 차단 여부 |
