# Notification (알림) 비즈니스 규칙

> 기획자 전달 알림 명세 표(2026-07-23)를 SSOT로 반영. 푸시·카카오 공유·정기 리마인드 전부 **wave 3** (2026-07-23 재분류 — 아래 "확정" 참고).

## 알림 이벤트 요약

| 규칙 ID | 이벤트 | 수신 대상 | 발송 조건 |
| :--- | :--- | :--- | :--- |
| **BR-NOTI-001** | 참여 완료 | 방장 | 참여자가 일정 제출을 완료한 경우 |
| **BR-NOTI-002** | 모든 참여자 일정 제출 완료 | 방장 | 여행방 **정원**(BR-TRIP-001, 1~10)에 도달한 순간(멤버는 join 즉시 RESPONDED이므로 "정원 도달"이 곧 전원 제출 완료) |
| **BR-NOTI-003** | 여행 정보 변경 | 참여자(방장 제외) | 방장이 여행방을 수정해 **실제 값이 하나 이상 바뀐 경우**(no-op patch는 미발송) (BR-TRIP-009·010) |
| **BR-NOTI-004** | 일정 확정 | 참여자(방장 제외) | 방장이 후보 일정 중 하나를 최종 일정으로 확정한 경우 (BR-TRIP-007) |
| **BR-NOTI-009** | 일정 확정 취소 | 참여자(방장 제외) | 방장이 확정 일정을 취소한 경우 (BR-TRIP-007) |
| **BR-NOTI-005** | 일정 업데이트 리마인드 (스케줄러) | 전체 사용자 | 매월 1일·15일 정기 발송 |

## 알림 문구·랜딩 화면

**BR-NOTI-001 · 참여 완료**
> OO님이 여행방에 참여했어요! 참여 현황을 확인해보세요.

랜딩 화면: 여행방 상세 · `BR-USER-005` Off 시 미발송

**BR-NOTI-002 · 모든 참여자 일정 제출 완료**
> 모든 참여자의 일정이 제출되었어요! 추천 일정을 받아보세요.

랜딩 화면: 여행방 상세 · `BR-USER-005` Off 시 미발송

**BR-NOTI-003 · 여행 정보 변경**
> 여행 정보가 변경되었어요. 변경된 내용을 확인해보세요.

랜딩 화면: 여행방 상세 · `BR-USER-005` Off 시 미발송

**BR-NOTI-004 · 일정 확정**
> 여행 일정이 확정되었어요! 확정된 일정을 확인해보세요.

랜딩 화면: 여행방 상세 · `BR-USER-005` Off 시 미발송

**BR-NOTI-009 · 일정 확정 취소**
> 확정된 여행 일정이 취소되었어요. 다시 일정을 조율해보세요.

랜딩 화면: 여행방 상세 · `BR-USER-005` Off 시 미발송

**BR-NOTI-005 · 일정 업데이트 리마인드**
> N월 일정을 업데이트해보세요. 더 정확한 여행 일정을 추천받을 수 있어요.

랜딩 화면: 일정 관리 · `BR-USER-005` Off 시 미발송

### 확정 (2026-07-23)

- 알림 문구·발송 조건·랜딩 화면을 기획자 전달 표 기준으로 반영
- BR-NOTI-004(확정·취소 통합)를 **확정(BR-NOTI-004)** / **취소(BR-NOTI-009 신설)** 로 분리 — 카피·랜딩이 서로 다름
- 구 BR-NOTI-006/007(카카오 공유·카피 예시)은 본문 표(001/002)에 통합되며 폐기. BR-NOTI-007 문구 "일정이 준비되었어요" → "일정이 제출되었어요"로 교체
- **BR-NOTI-005를 wave 4 → wave 3로 재분류** (`docs/specs/notification.md` D1, 사용자 승인) — `#21`/`#31`/`#32`·`development-wave.md`·`waves.md`·`mvp.md` 동시 amend
- **BR-USER-005(알림 on/off) 확정** (`docs/specs/notification.md` D2) — `user.md` BR-USER-005 참고. **예외 없이 NOTI-001~005·009 전체에 게이트 적용** (D10)
- **NOTI-003/004/009 수신 대상 "참여자(방장 제외)"로 확정** (`docs/specs/notification.md` D3) — 액션을 수행한 방장 본인은 제외
- **NOTI-002 판정 기준 확정**: 여행방 정원 도달 순간 (`docs/specs/notification.md` D11)
- **NOTI-003 발송 조건 확정**: patchTrip이 실제 값 변경을 수반할 때만 발송, no-op은 미발송 (`docs/specs/notification.md` D12)
- **NOTI-005 발송 방식 확정**: FCM 토픽이 아닌 **DB 조회 + 배치(500개) 멀티캐스트** — `notification_enabled=true`인 사용자만 대상, 게이트를 서버가 완전히 통제 (`docs/specs/notification.md` D6)
